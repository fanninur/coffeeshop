# AGENTS.md

# Project Overview

Build and maintain a modern premium coffee shop website for:

# TUDEY COFFEE

The website should represent a lifestyle café that is warm, modern, and welcoming for young people.

The primary goal is not only to showcase products, but to create a digital experience that encourages visitors to come to the café, explore the menu, and become repeat customers.

---

# Project Mission

Every design and development decision should reinforce these values:

* Warm atmosphere
* Great coffee
* Community
* Lifestyle
* Simplicity
* Quality
* Comfort

Never sacrifice usability for visual effects.

---

# Design Philosophy

The website should feel like:

> Walking into a cozy modern café with the aroma of fresh coffee.

Every interaction should feel:

* warm
* elegant
* effortless
* smooth
* inviting

Avoid designs that feel:

* corporate
* futuristic
* overly minimal
* luxury restaurant
* template-based
* dark industrial

---

# Target Audience

Primary users:

* Students
* Young professionals
* Freelancers
* Coffee lovers
* Remote workers
* Couples
* Small communities

Age:

18–35

---

# Website Goals

The website should help users:

* Discover the café
* Explore the menu
* View promotions
* Find the location
* Contact the café
* Order quickly
* Follow social media

Every page should guide users toward these actions.

---

# Technology Stack

Preferred:

* Astro
* Tailwind CSS
* TypeScript
* Component-based architecture

Optional:

* GSAP
* Lenis
* SwiperJS
* Framer Motion

Avoid adding unnecessary dependencies.

Performance always has priority over visual complexity.

---

# Development Principles

Always:

* Build reusable components.
* Keep layouts modular.
* Write clean code.
* Follow accessibility standards.
* Optimize for performance.
* Prefer semantic HTML.
* Optimize images.
* Use responsive layouts.

Never:

* Duplicate code.
* Create one-off components unless necessary.
* Use inline CSS unnecessarily.
* Install heavy libraries for simple interactions.

---

# Design System Rules

Before creating a new section:

Ask yourself:

* Does this improve the customer journey?
* Does it strengthen the brand?
* Is it mobile-friendly?
* Is it visually balanced?
* Is it consistent with the existing design?

If the answer is "No", redesign it.

---

# UI Style

The UI should feel:

* warm
* clean
* modern
* friendly
* premium
* spacious

Use:

* rounded corners
* soft shadows
* warm colors
* subtle hover states
* large product imagery
* comfortable spacing

Avoid:

* sharp corporate layouts
* excessive borders
* flashy gradients
* excessive blur
* overly complex interfaces

---

# Animation Rules

Animations should enhance the experience, never distract.

Allowed:

* Fade-in
* Fade-up
* Staggered reveal
* Hover lift
* Image zoom
* Smooth scrolling
* Button micro-interactions

Avoid:

* Heavy parallax
* Complex 3D scenes
* Long loading animations
* Flashing effects
* Excessive motion

Animations should remain smooth on mid-range mobile devices.

---

# Image Guidelines

Prioritize images that show:

* Coffee preparation
* Latte art
* Signature drinks
* Fresh pastries
* Café ambience
* Customers enjoying the space
* Natural lighting
* Lifestyle moments

Avoid:

* Generic stock office photos
* Corporate business meetings
* Luxury restaurant settings
* Overly edited images
* Artificial-looking AI imagery

---

# Component Guidelines

Create reusable components such as:

* Navbar
* Hero
* SectionTitle
* MenuCard
* PromoCard
* FeatureCard
* GalleryGrid
* TestimonialCard
* CTA
* Footer

Each component must:

* be reusable
* accept props
* remain independent
* support responsive layouts

---

# Layout Rules

Use generous spacing.

Preferred section spacing:

* 96–160px vertically on desktop
* 64–96px on tablet
* 48–72px on mobile

Content width:

* Maximum 1200–1280px
* Center aligned

Avoid overcrowded layouts.

---

# Typography Rules

Use:

* Plus Jakarta Sans
* Inter

Headings should be:

* bold
* expressive
* highly readable

Body text should be:

* concise
* relaxed
* easy to scan

Never use decorative fonts.

---

# Button Rules

Primary buttons:

* Warm caramel color
* Rounded
* Clear hover state
* Strong contrast

Secondary buttons:

* Outline style
* Brown border
* Subtle hover background

Buttons should always have visible hover and focus states.

---

# Mobile-First Rules

Assume most visitors use smartphones.

Prioritize:

* Thumb-friendly navigation
* Large touch targets
* Fast loading
* Optimized images
* Sticky CTA when appropriate
* Simple menus

Desktop should enhance the experience, not redefine it.

---

# Accessibility

Ensure:

* Semantic HTML
* Proper heading hierarchy
* Alt text on all images
* Keyboard navigation
* Visible focus states
* Sufficient color contrast

Accessibility is not optional.

---

# Performance

Target:

* Lighthouse Performance ≥ 90
* Accessibility ≥ 95
* Best Practices ≥ 95
* SEO ≥ 95

Always:

* Lazy-load images
* Compress assets
* Minimize JavaScript
* Avoid layout shifts
* Optimize fonts

---

# SEO

Use semantic HTML.

Optimize:

* Meta title
* Meta description
* Open Graph
* Structured headings
* Image alt text

Focus keywords:

* coffee shop
* café
* coffee near me
* coffee shop [city]
* tempat nongkrong
* kopi susu
* café aesthetic

---

# Coding Standards

Keep code:

* modular
* readable
* maintainable
* scalable

Comment only when necessary.

Avoid unnecessary abstraction.

---

# Before Every Commit

Confirm:

* No broken layouts
* Responsive on all breakpoints
* Images optimized
* Accessibility maintained
* No console errors
* No unused imports
* Consistent spacing
* Consistent typography
* Consistent colors

---

# Final Experience Goal

Every visitor should immediately think:

> "This café looks cozy, the coffee looks amazing, and I want to visit."

The website should feel like an extension of the café itself—not just an online menu.

Every decision should support this experience.
